## 1. Регистрация и первый вход

### Шаг 1.1. Откройте сайт

1. Перейдите на **[cloud.yandex.ru](https://cloud.yandex.ru)**.
2. В правом верхнем углу нажмите **Войти** или **Начать использовать**.

### Шаг 1.2. Вход через Яндекс ID

Яндекс Облако использует единый аккаунт Яндекс:

- Введите **логин** (email или телефон) и **пароль** от Яндекс ID.
- Если аккаунта нет — нажмите **Зарегистрироваться** и создайте его.
- При первом входе примите условия использования облака.

### Шаг 1.3. Стартовый грант (если доступен)

- Новым пользователям иногда дают **стартовый грант** на тестирование (проверьте актуальность на сайте).
- Баланс отображается в правом верхнем углу (иконка кошелька или сумма).
- Если гранта нет — потребуется **привязать банковскую карту** для оплаты. Списание идёт по факту использования (pay-as-you-go).

### Шаг 1.4. Консоль управления

После входа откроется **Консоль управления** — главная страница с:

- **Левое меню**: Compute Cloud, VPC, и другие сервисы.
- **Верхняя панель**: выбор организации, каталога, уведомления, профиль.
- **Центральная область**: обзор ресурсов и быстрые действия.

## 2. Создание каталога

Каталог (folder) — контейнер для ресурсов: ВМ, сетей, дисков и т.д. В одном каталоге удобно держать все ресурсы проекта.

### Шаг 2.1. Открыть выбор каталога

1. В **левом верхнем углу** найдите выпадающий список с названием каталога (или организации).
2. Нажмите на него — откроется меню с каталогами.

### Шаг 2.2. Создать новый каталог

1. В выпадающем меню выберите **Создать каталог**.
2. Откроется форма создания.

### Шаг 2.3. Заполнение формы

| Поле | Значение | Описание |
|------|----------|----------|
| **Имя** | `mathcalc` | Имя каталога (латиница, без пробелов). Отображается в консоли. |
| **Описание** | `MathCalc backend` (опционально) | Краткое описание для себя. |

### Шаг 2.4. Создание

1. Нажмите **Создать**.
2. Каталог появится в списке.
3. Выберите его — он станет **активным** (текущие действия будут выполняться в этом каталоге).

## 3. Создание виртуальной машины

### Шаг 3.1. Открыть Compute Cloud

1. В **левом меню** найдите **Compute Cloud**.
2. Нажмите на него — откроется подменю.
3. Выберите **Виртуальные машины**.
4. Откроется список ВМ (пока пустой).

### Шаг 3.2. Создать ВМ

1. Нажмите синюю кнопку **Создать ВМ** (в правом верхнем углу или по центру).
2. Откроется многошаговая форма создания.

### Шаг 3.3. Основные параметры (шаг 1)

| Поле | Значение | Описание |
|------|----------|----------|
| **Имя** | `mathcalc-backend` | Имя ВМ в консоли. Можно изменить позже. |
| **Зона доступности** | `ru-central1-a` | Регион Москва, зона A. Альтернативы: `ru-central1-b`, `ru-central1-d`. |
| **Платформа** | Intel Ice Lake | Стандартная платформа. Оставьте по умолчанию. |

### Шаг 3.4. Вычислительные ресурсы

В блоке **Вычислительные ресурсы**:

| Ресурс | Рекомендуется | Минимум (тест) | Где менять |
|--------|---------------|----------------|------------|
| **Платформа** | Intel Ice Lake | Intel Ice Lake | Выпадающий список |
| **Гарантированная доля vCPU** | 100% | 100% | Ползунок или выбор |
| **vCPU** | 2 | 1 | Выпадающий список |
| **RAM** | 2 ГБ | 1 ГБ | Выпадающий список |

Для NestJS достаточно **2 vCPU, 2 ГБ RAM**. Для экономии можно взять 1 vCPU, 1 ГБ.

### Шаг 3.5. Загрузочный диск

В блоке **Диски**:

1. Нажмите **Добавить диск** (если диск ещё не добавлен).
2. Выберите **Создать диск**.
3. Заполните:

| Поле | Значение | Описание |
|------|----------|----------|
| **Имя** | `mathcalc-boot` | Имя диска |
| **Размер** | 10 ГБ | Минимум для Ubuntu. Можно 15–20 ГБ. |
| **Тип диска** | SSD | Быстрее. HDD — дешевле. |
| **Образ** | Ubuntu 22.04 LTS | Выберите в списке образов. Ищите "Ubuntu" → "22.04 LTS". |

4. Убедитесь, что диск помечен как **Загрузочный** (boot disk).

### Шаг 3.6. Сетевые настройки

В блоке **Сетевые настройки**:

| Поле | Значение | Описание |
|------|----------|----------|
| **Сеть** | `default` | Стандартная сеть. Или выберите свою. |
| **Подсеть** | Подсеть в `ru-central1-a` | Должна соответствовать зоне ВМ. |
| **Публичный адрес** | **Автоматически** | Обязательно! Без этого к ВМ нельзя подключиться из интернета. |
| **Внутренний адрес** | Автоматически | Можно оставить по умолчанию. |

### Шаг 3.7. Доступ (SSH-ключ)

Разверните блок **Доступ** (или **Доступ к ВМ**):

| Поле | Значение | Описание |
|------|----------|----------|
| **Сервисный аккаунт** | Не назначен | Оставьте пустым для простого сценария. |
| **Логин** | `ubuntu` | Стандартный пользователь для Ubuntu. Не меняйте. |
| **SSH-ключ** | Ваш публичный ключ | См. шаг 3.8. |

### Шаг 3.8. Получение SSH-ключа

**На вашем Mac или Linux** откройте терминал и выполните:

```bash
# Проверка: есть ли уже ключ
ls ~/.ssh/id_ed25519.pub
```

Если файл есть — скопируйте его содержимое:

```bash
cat ~/.ssh/id_ed25519.pub
```

Если файла нет — создайте ключ:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com" -f ~/.ssh/id_ed25519 -N ""
```

Флаг `-N ""` — пустая парольная фраза (для автоматизации). При желании можно задать пароль.

Скопируйте вывод `cat ~/.ssh/id_ed25519.pub` — строка вида:

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... your_email@example.com
```

### Шаг 3.9. Вставка SSH-ключа в форму

1. В поле **SSH-ключ** вставьте скопированную строку целиком.
2. Можно добавить несколько ключей — нажмите **Добавить ключ** для каждого.

### Шаг 3.10. Создание ВМ

1. Прокрутите вниз и нажмите **Создать ВМ**.
2. Дождитесь создания (обычно 1–2 минуты).
3. В списке ВМ появится новая строка со статусом **Running**.
4. В колонке **Публичный IPv4** будет адрес вида `84.201.xxx.xxx` — **сохраните его**.

## 4. Подключение по SSH

### Шаг 4.1. Открыть терминал

На вашем Mac или Linux откройте **Терминал** (Terminal, iTerm и т.п.).

### Шаг 4.2. Команда подключения

```bash
ssh ubuntu@<ПУБЛИЧНЫЙ_IP_ВМ>
```

Замените `<ПУБЛИЧНЫЙ_IP_ВМ>` на IP из шага 3.10.

**Пример:**
```bash
ssh ubuntu@84.201.123.45
```

### Шаг 4.3. Первый вход — подтверждение fingerprint

При первом подключении появится:

```
The authenticity of host '84.201.123.45 (84.201.123.45)' can't be established.
ED25519 key fingerprint is SHA256:xxxxx...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

Введите `yes` и нажмите Enter.

### Шаг 4.4. Проверка

Вы должны оказаться в shell на сервере. Приглашение будет вида:

```
ubuntu@mathcalc-backend:~$
```

Теперь все команды выполняются **на сервере**, а не на вашем компьютере.

## 5. Установка Node.js

### Шаг 5.1. Установка nvm

Выполните на сервере (после `ssh`):

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

Ожидаемый вывод: сообщение об успешной установке nvm.

### Шаг 5.2. Загрузка nvm в текущую сессию

nvm добавляет себя в `~/.bashrc`, но текущая сессия его ещё не видит:

```bash
source ~/.bashrc
```

Если при подключении по SSH используется zsh:

```bash
source ~/.zshrc
```

### Шаг 5.3. Проверка nvm

```bash
nvm --version
```

Должна отобразиться версия (например, `0.39.7`).

### Шаг 5.4. Установка Node.js 20

```bash
nvm install 20
nvm use 20
nvm alias default 20
```

### Шаг 5.5. Проверка

```bash
node -v   # v20.x.x
npm -v    # 10.x.x
```

Если версии отображаются — Node.js установлен.

## 6. Клонирование и сборка проекта

### Шаг 6.1. Установка Git

```bash
sudo apt update
sudo apt install -y git
```

### Шаг 6.2. SSH-ключ для GitHub (если репозиторий приватный)

Если репозиторий **приватный**, нужен отдельный ключ для GitHub:

```bash
ssh-keygen -t ed25519 -C "your_email@example.com" -f ~/.ssh/id_github -N ""
cat ~/.ssh/id_github.pub
```

Скопируйте вывод и добавьте в GitHub:

1. GitHub → **Settings** → **SSH and GPG keys** → **New SSH key**
2. Вставьте ключ, сохраните.

Настройте использование этого ключа для GitHub:

```bash
echo 'Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_github' >> ~/.ssh/config
chmod 600 ~/.ssh/config
```

Если репозиторий **публичный** — можно использовать HTTPS и пропустить настройку SSH для GitHub.

### Шаг 6.3. Клонирование репозитория

**Через SSH** (если настроили ключ):

```bash
cd ~
git clone git@github.com:Edgarok1991/math-stat-calculator.git
cd math-stat-calculator
```

**Через HTTPS** (проще для публичных репозиториев):

```bash
cd ~
git clone https://github.com/Edgarok1991/math-stat-calculator.git
cd math-stat-calculator
```

### Шаг 6.4. Переход в backend

```bash
cd calculator/backend
```

### Шаг 6.5. Установка зависимостей

```bash
npm ci
```

`npm ci` устанавливает точные версии из `package-lock.json` (рекомендуется для production).

### Шаг 6.6. Создание .env

Перед сборкой нужно создать файл `.env` (см. раздел 7). Пока создайте пустой и заполните позже, или сразу вставьте содержимое:

```bash
nano .env
```

Вставьте содержимое из раздела 7, сохраните (`Ctrl+O`, `Enter`, `Ctrl+X`).

### Шаг 6.7. Prisma и сборка

```bash
npx prisma generate
npx prisma db push
npm run build
```

### Шаг 6.8. Ожидаемый вывод

При успехе:

```
✔ Generated Prisma Client
✔ The database is now in sync with your schema.
...
✔ Built successfully
```

Папка `dist/` должна появиться с скомпилированным кодом.

## 7. Настройка .env на сервере

### Шаг 7.1. Открыть .env в редакторе

```bash
nano ~/math-stat-calculator/calculator/backend/.env
```

### Шаг 7.2. Шаблон содержимого

Скопируйте и **подставьте свои значения** (замените плейсхолдеры):

```env
# Supabase (скопируйте из локального calculator/backend/.env)
DATABASE_URL="postgres://prisma.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:5432/postgres"
DIRECT_URL="postgres://prisma.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:5432/postgres"

# JWT (сгенерируйте НОВЫЙ для production!)
JWT_SECRET="ваш-длинный-случайный-ключ-минимум-32-символа"
JWT_EXPIRATION="7d"

# App
PORT=3001
NODE_ENV=production

# Frontend (URL вашего проекта на Vercel)
FRONTEND_URL="https://ваш-проект.vercel.app"
APP_URL="https://ваш-проект.vercel.app"

# SMTP Yandex (id.yandex.ru → Безопасность → Пароли приложений)
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=587
SMTP_USER=ваш_email@yandex.ru
SMTP_PASS=пароль_приложения_яндекс
```

### Шаг 7.3. Генерация JWT_SECRET

На сервере или локально:

```bash
openssl rand -base64 32
```

Скопируйте вывод (например, `K7x9mP2vL4nQ8wR1tY3uI6oE0aB5cD...`) и вставьте в `JWT_SECRET`.

### Шаг 7.4. Сохранение в nano

- `Ctrl+O` — сохранить
- `Enter` — подтвердить имя файла
- `Ctrl+X` — выйти

## 8. Запуск через PM2

PM2 — менеджер процессов для Node.js. Держит приложение запущенным и перезапускает при сбоях.

### Шаг 8.1. Установка PM2 глобально

```bash
sudo npm install -g pm2
```

### Шаг 8.2. Запуск приложения

```bash
cd ~/math-stat-calculator/calculator/backend
pm2 start dist/main.js --name mathcalc-api
```

### Шаг 8.3. Ожидаемый вывод

```
[PM2] Starting dist/main.js in fork_mode (1 instance)
[PM2] Done.
┌─────┬──────────────┬─────────────┬─────────┬─────────┬──────────┐
│ id  │ name         │ mode        │ status  │ cpu     │ memory   │
├─────┼──────────────┼─────────────┼─────────┼─────────┼──────────┤
│ 0   │ mathcalc-api │ fork        │ online  │ 0%      │ 50.0mb   │
└─────┴──────────────┴─────────────┴─────────┴─────────┴──────────┘
```

Статус `online` — приложение запущено.

### Шаг 8.4. Автозапуск при перезагрузке сервера

```bash
pm2 startup
```

PM2 выведет команду вида:

```
sudo env PATH=... pm2 startup systemd -u ubuntu --hp /home/ubuntu
```

**Скопируйте и выполните эту команду целиком** (она уникальна для вашей системы).

Затем сохраните текущий список процессов:

```bash
pm2 save
```

### Шаг 8.5. Проверка логов

```bash
pm2 logs mathcalc-api
```

Должны быть строки вида `Backend сервер запущен на порту 3001`. Для выхода: `Ctrl+C`.

## 9. Открытие порта в firewall

По умолчанию входящий трафик на порт 3001 может быть заблокирован. Нужно разрешить его.

### Вариант A: Группа безопасности в консоли Яндекс Облака

1. В **левом меню** выберите **VPC** → **Группы безопасности**.
2. Найдите группу, связанную с вашей ВМ (часто `default`).
3. Нажмите на неё.
4. Перейдите в **Правила**.
5. Нажмите **Добавить правило** (входящий трафик).
6. Заполните:
   - **Порт**: `3001`
   - **Протокол**: TCP
   - **Источник**: `0.0.0.0/0` (любой IP) или `CIDR` с вашим IP
7. Сохраните.

### Вариант B: UFW на сервере

Если группа безопасности уже разрешает трафик, но UFW включён:

```bash
sudo ufw allow 3001
sudo ufw allow 22
sudo ufw enable
```

При запросе подтверждения введите `y`.

Проверка:

```bash
sudo ufw status
```

## 10. Проверка работы

### Шаг 10.1. Проверка с вашего компьютера

Откройте в браузере:

```
http://<ПУБЛИЧНЫЙ_IP_ВМ>:3001
```

Пример: `http://84.201.123.45:3001`

Должен вернуться JSON (например, приветствие API или список маршрутов).

### Шаг 10.2. Проверка через curl

В терминале на вашем компьютере:

```bash
curl http://<ПУБЛИЧНЫЙ_IP_ВМ>:3001
```

### Шаг 10.3. Настройка Vercel

1. Откройте [vercel.com](https://vercel.com) → ваш проект.
2. **Settings** → **Environment Variables**.
3. Добавьте:
   - **Name**: `NEXT_PUBLIC_API_URL`
   - **Value**: `http://<ПУБЛИЧНЫЙ_IP_ВМ>:3001`
   - **Environment**: Production (и Preview при необходимости)
4. Сохраните.
5. **Deployments** → **Redeploy** последнего деплоя.

После redeploy frontend на Vercel будет обращаться к backend в Яндекс Облаке.

## 11. Nginx (опционально)

Если нужен доступ по порту 80 (без `:3001`) или по домену.

### Шаг 11.1. Установка Nginx

```bash
sudo apt install -y nginx
```

### Шаг 11.2. Создание конфигурации

```bash
sudo nano /etc/nginx/sites-available/mathcalc
```

Вставьте (замените `YOUR_IP` на публичный IP или `your-domain.ru`):

```nginx
server {
    listen 80;
    server_name YOUR_IP;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Шаг 11.3. Активация

```bash
sudo ln -s /etc/nginx/sites-available/mathcalc /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### Шаг 11.4. Открытие порта 80

В группе безопасности или UFW добавьте правило для порта 80. Тогда API будет доступен по `http://<IP>` без `:3001`.

## 13. Частые ошибки и решения

### Ошибка: "Permission denied (publickey)"

**Причина**: SSH-ключ не добавлен в ВМ или неверный.

**Решение**:
- Проверьте, что публичный ключ добавлен при создании ВМ.
- В консоли: ВМ → **Остановить** → **Изменить** → раздел **Доступ** → добавьте ключ заново.
- Убедитесь, что используете правильный ключ: `ssh -i ~/.ssh/id_ed25519 ubuntu@<IP>`.

---

### Ошибка: "Connection refused" при обращении к API

**Причина**: Порт 3001 не открыт или backend не запущен.

**Решение**:
- На сервере: `pm2 status` — процесс должен быть `online`.
- Проверьте группу безопасности в консоли — правило для порта 3001.
- Проверьте UFW: `sudo ufw status`.

---

### Ошибка: "Can't reach database server" (Prisma)

**Причина**: Supabase недоступен с сервера.

**Решение**:
- Проверьте `.env` — правильный ли `DATABASE_URL` и `DIRECT_URL`.
- Убедитесь, что проект Supabase не приостановлен (Paused) — восстановите в Dashboard.
- Используйте Session pooler (порт 5432), а не Direct.

---

### PM2 не перезапускается после reboot

**Решение**: Выполните снова:
```bash
pm2 startup
# Выполните выведенную команду
pm2 save
```

---

### Ошибка: "EADDRINUSE: address already in use"

**Причина**: Порт 3001 уже занят другим процессом.

**Решение**:
```bash
pm2 delete mathcalc-api
pm2 start dist/main.js --name mathcalc-api
```

---

### Ошибка: "npm: command not found" после установки nvm

**Причина**: nvm не загружен в текущую сессию.

**Решение**:
```bash
source ~/.bashrc
nvm use 20
```

## Сводка: URL backend

После настройки backend доступен по адресу:

- **Без Nginx**: `http://<ПУБЛИЧНЫЙ_IP>:3001`
- **С Nginx**: `http://<ПУБЛИЧНЫЙ_IP>` или `http://ваш-домен.ru`

Этот URL указывается в Vercel как `NEXT_PUBLIC_API_URL`.
