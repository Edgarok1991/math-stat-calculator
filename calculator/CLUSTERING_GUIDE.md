# 🔍 Кластерный анализ - Руководство пользователя

## Описание

Калькулятор кластерного анализа выполняет классификацию объектов методами **ближнего соседа (Single Linkage)** и **дальнего соседа (Complete Linkage)** с построением детальных таблиц и дендрограммы.

Реализация основана на примере с сайта: https://axd.semestr.ru/upr/cluster.php

## 🎯 Возможности

### Методы кластеризации

1. **Ближний сосед (Single Linkage)**
   - Расстояние между кластерами = минимальное расстояние между их элементами
   - Подходит для вытянутых кластеров

2. **Дальний сосед (Complete Linkage)**
   - Расстояние между кластерами = максимальное расстояние между их элементами
   - Подходит для компактных кластеров

3. **Средний (Average Linkage)**
   - Расстояние между кластерами = среднее расстояние между всеми парами элементов
   - Универсальный метод

4. **K-means**
   - Быстрый итеративный метод
   - Для сферических кластеров

### Детальное пошаговое решение

Калькулятор показывает:

✅ **Матрицы расстояний на каждом шаге**
- Полная матрица с выделением минимальных расстояний
- Цветовое кодирование объединяемых кластеров

✅ **Пошаговые объяснения**
- Какие кластеры объединяются
- Почему выбрана данная пара
- Как формируется новая матрица

✅ **Дендрограмма**
- Визуализация иерархии объединения
- Отображение расстояний на каждом уровне

✅ **Итоговые результаты**
- Количество кластеров
- Состав каждого кластера
- Расстояние между финальными кластерами

## 📋 Формат ввода данных

### Способ 1: Построчный ввод

Каждая строка = один объект. Координаты разделяются запятыми.

```

4, 10
5, 7
12, 6
14, 6
15, 4
```

## 📊 Интерпретация результатов

### Матрица расстояний

- **Зеленая ячейка** - минимальное расстояние (пара для объединения)
- **Желтый фон** - строки/столбцы объединяемых кластеров
- **Красный фон** - точная пара минимального расстояния

### Пошаговое решение

Каждый шаг показывает:
1. Текущую матрицу расстояний
2. Найденное минимальное расстояние
3. Какие кластеры объединяются
4. Как формируется новая матрица
5. Количество оставшихся кластеров

### Дендрограмма

- **Ось X** - объекты
- **Ось Y** - расстояние объединения
- **Красные числа** - расстояние на момент объединения кластеров
- **Высота линий** - мера различия между кластерами

## 💡 Советы по использованию

### Для получения детального решения

✅ Выбирайте **иерархические методы** (ближний/дальний/средний сосед)
❌ K-means не показывает пошаговые таблицы

### Оптимальное количество объектов

- **Минимум**: 2 объекта
- **Рекомендуется**: 4-10 объектов для наглядности
- **Максимум**: до 100 объектов (но таблицы станут большими)

### Выбор метода

- **Ближний сосед** - для поиска цепочек и вытянутых групп
- **Дальний сосед** - для компактных, хорошо разделённых групп
- **Средний** - компромисс между ними
- **K-means** - когда нужна скорость, а не детали

## 🔄 Расчёт расстояний

Используется **Евклидово расстояние**:

```
d(i,j) = √[(x₁ᵢ - x₁ⱼ)² + (x₂ᵢ - x₂ⱼ)² + ... + (xₙᵢ - xₙⱼ)²]
```

Для кластеров:

- **Single Linkage**: `d(A,B) = min{d(a,b) : a ∈ A, b ∈ B}`
- **Complete Linkage**: `d(A,B) = max{d(a,b) : a ∈ A, b ∈ B}`
- **Average Linkage**: `d(A,B) = avg{d(a,b) : a ∈ A, b ∈ B}`

## 🚀 Технические детали

### Backend (NestJS)

- **Эндпоинт**: `POST /clustering/calculate`
- **Вход**: `{ points: number[][], k: number, method: string }`

- **Компоненты**:
  - `ClusteringStepsView` - пошаговые таблицы

  - `ClusteringPage` - главная страница

- **Framer Motion** - анимация
- **React Hook Form + Zod** - валидация форм
- **Canvas API** - отрисовка дендрограммы
