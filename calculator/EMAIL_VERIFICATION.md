# 📧 Email-верификация - Руководство

## ✅ Что реализовано

### Продвинутая регистрация с подтверждением email!

**Workflow:**
1. Пользователь регистрируется
2. На email отправляется письмо с ссылкой
3. Пользователь переходит по ссылке
4. Email подтверждается
5. Автоматический вход в систему
6. Приветственное письмо

---

## 🔧 Технические детали

### База данных (обновлено)

**Добавлены поля в таблицу users:**
- `emailVerified` (boolean) - подтверждён ли email
- `verificationToken` (string, unique) - токен для подтверждения

### Email сервис

**Используется:**
- **Nodemailer** - отправка писем
- **Ethereal Email** - тестовый SMTP для разработки
- **Красивые HTML письма** с градиентами

### API Endpoints

```
POST /auth/register           - регистрация (отправляет письмо)
GET  /auth/verify?token=...   - подтверждение email
POST /auth/resend-verification - повторная отправка письма
```

## 🔒 Безопасность

### Проверки при входе

```typescript
if (!user.emailVerified) {
  throw new Error('Email не подтверждён');
}
```

**Нельзя войти без подтверждения email!**

### Токен верификации

- 32 байта случайных данных
- Хранится в БД
- Уникальный для каждого пользователя
- Удаляется после подтверждения

## 💡 Режим разработки

### Ethereal Email

**Что это:**
- Тестовый SMTP сервер
- Не отправляет реальные письма
- Даёт ссылку для просмотра в браузере

**Как работает:**
1. Регистрируетесь в приложении
2. В консоли backend появляется:
   ```
   📧 Письмо отправлено
   🔗 Ссылка для просмотра: https://ethereal.email/message/...
   ```
3. Открываете ссылку
4. Видите красивое письмо
5. Копируете ссылку активации

### В Frontend

При регистрации показывается:
```
✅ Регистрация успешна!
📧 Письмо отправлено

🔧 Режим разработки:
[Открыть письмо в браузере →]
```

## 🚀 Production настройка

### Для реальной отправки писем

**Вариант 1: Gmail**

```typescript
const transporter = nodemailer.createTransport({
  service: 'gmail',
  auth: {
    user: process.env.GMAIL_USER,
    pass: process.env.GMAIL_APP_PASSWORD,
  },
});
```

**.env:**
```env
GMAIL_USER=your@gmail.com
GMAIL_APP_PASSWORD=your-app-password
```

**Вариант 2: SendGrid**

```bash
npm install @sendgrid/mail
```

```typescript
import sgMail from '@sendgrid/mail';
sgMail.setApiKey(process.env.SENDGRID_API_KEY);
```

**Вариант 3: AWS SES**

```bash
npm install @aws-sdk/client-ses
```
